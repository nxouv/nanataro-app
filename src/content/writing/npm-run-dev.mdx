プログラミングを始めて最初に覚えたコマンドが何だったか、覚えていますか。僕の場合はnpm run devでした。

正直なところ、最初はこのコマンドが何をしているのか全くわかっていませんでした。チュートリアルに「ターミナルでnpm run devと入力してください」と書いてあったので、言われるがままに入力していただけです。

## ターミナルという未知の領域

ターミナルを開くこと自体が最初のハードルでした。Macの場合はSpotlightで「ターミナル」と検索すれば出てくるのですが、そもそもSpotlightの存在も知らなかったので、アプリケーションフォルダを延々とスクロールしていた記憶があります。

黒い画面（実際は白かったけど）に白い文字が並んでいるのを見て、「これは自分には無理かもしれない」と思いました。でも、とりあえず書いてある通りにやってみることにしました。

## 最初のエラー

案の定、最初はうまくいきませんでした。

当時の僕にはエラーメッセージが何を意味しているのか全くわかりませんでした。でも検索してみると、同じエラーで困っている人がたくさんいて、解決方法も見つかりました。

## 学んだこと

この経験から学んだことがいくつかあります。

- エラーメッセージはちゃんと読めば解決のヒントが書いてある
- わからないことは検索すれば大抵答えが見つかる
- 最初は誰でもわからないことだらけ

今でもエラーが出るとドキッとしますが、最初の頃に比べればだいぶ慣れました。

npm run devを覚えた日から、少しずつですが前に進んでいる気がします。
